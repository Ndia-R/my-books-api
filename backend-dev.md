# ⚙️ バックエンド開発環境ガイド (my-books-api)

本ガイドは、Spring Boot ベースのバックエンドサービス（`my-books-api`）をローカルで開発するための環境設定と利用方法を説明します。Docker Compose を利用することで、アプリケーションと依存するデータベース（MySQL）を一括で起動します。

## 🚀 1. 環境構成の概要

本環境は、以下の 2 つのコンテナで構成されます。

| サービス名         | 役割                                 | Dockerfile ステージ | 公開ポート                              | 依存関係      |
| :----------------- | :----------------------------------- | :------------------ | :-------------------------------------- | :------------ |
| **`my-books-api`** | バックエンド API 本体（Spring Boot） | `development`       | ホスト 8080 $\rightarrow$ コンテナ 8080 | `my-books-db` |
| **`my-books-db`**  | 開発用データベース（MySQL 8.0）      | -                   | ホスト 3306 $\rightarrow$ コンテナ 3306 | -             |

## 🛠️ 2. 開発環境の特長

### 2-1. 高速な開発サイクル

| 機能                       | 詳細                                                                                                                                                                                                    |
| :------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **ソースコードの即時反映** | ホスト側のソースコード (`.`) がコンテナ内の `/workspace` にマウントされているため、コードの変更が即座に反映されます（ホットリロードの設定が必要です）。                                                 |
| **Gradle キャッシュ**      | Gradle の依存関係キャッシュが Volume (`gradle-cache`) として永続化されるため、再ビルド時の依存関係ダウンロード時間を大幅に短縮できます。                                                                |
| **多機能な開発イメージ**   | `Dockerfile` の `development` ステージには、**JDK 21** に加えて **Git**, **Node.js**, **Python (uv)**、**Gemini CLI** などがプリインストールされており、AI 連携や各種開発タスクをコンテナ内で行えます。 |

### 2-2. データベースの管理

- **初期データ自動投入**: `docker-compose.yml` に定義された各種 `.csv` および `init.sql` がコンテナ起動時に自動でインポートされ、すぐに開発可能な状態になります。
- **ヘルスチェック**: `my-books-api` は、**`my-books-db` のヘルスチェック**に合格するまで起動を待機するため、データベース接続エラーを防ぎます。

## 🔑 3. 認証・外部接続設定

API は以下の環境変数を通じて外部サービスと連携します。

| 環境変数                | 役割                              | 設定値（例）                                  |
| :---------------------- | :-------------------------------- | :-------------------------------------------- |
| `SPRING_DATASOURCE_URL` | データベース接続 URL              | `jdbc:mysql://my-books-db:3306/db_name`       |
| **`IDP_ISSUER_URI`**    | **Keycloak の認証エンドポイント** | ローカル Keycloak の URL を設定してください。 |

> 💡 **注意**: 開発用の Keycloak との接続のために、`Dockerfile` の `development` ステージには **`mkcert` CA 証明書**が一時的に追加されています。

## 🏗️ 4. 本番デプロイ用イメージの作成

開発が完了したら、`Dockerfile` の**マルチステージビルド**を利用して、軽量でセキュアな本番環境用の JAR ファイルを含むイメージを作成します。

**ビルドコマンド例:**

```bash
# production ステージをターゲットに指定し、本番環境用のイメージをビルド
docker image build --target production -t <レジストリURL>/my-books-api:<タグ> .
```

```bash
# 例
docker image build --target production -t vsv-crystal.skygroup.local/my-books-api:latest .
```
